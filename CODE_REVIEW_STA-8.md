### 🟡 Рекомендации по улучшению

#### 1. **Производительность: Мемоизация вычислений**

**Проблема:** Функция `filteredTodos` вычисляется на каждый рендер компонента, даже когда зависимости не изменились.

**Текущий код:**
```typescript
const filteredTodos = todos
  .filter((todo) => {
    if (filterStatus === "active") return !todo.completed;
    if (filterStatus === "completed") return todo.completed;
    return true;
  })
  .filter((todo) =>
    todo.text.toLowerCase().includes(searchText.toLowerCase())
  );
```

**Рекомендация:** Использовать `useMemo` для оптимизации:

```typescript
const filteredTodos = useMemo(() => {
  return todos
    .filter((todo) => {
      if (filterStatus === "active") return !todo.completed;
      if (filterStatus === "completed") return todo.completed;
      return true;
    })
    .filter((todo) =>
      todo.text.toLowerCase().includes(searchText.toLowerCase())
    );
}, [todos, filterStatus, searchText]);
```

**Импакт:** Средний - улучшит производительность при большом количестве задач.

---

#### 2. **UX: Debounce для поиска**

**Проблема:** Поиск срабатывает на каждое нажатие клавиши, что может быть неэффективно для больших списков.

**Рекомендация:** Добавить debounce для поля поиска (300-500ms). Можно использовать библиотеку `use-debounce`:

```typescript
import { useDebounce } from 'use-debounce';

const [searchText, setSearchText] = useState("");
const [debouncedSearchText] = useDebounce(searchText, 300);

// Использовать debouncedSearchText в фильтрации
```

**Импакт:** Низкий для текущего состояния приложения, но важен для масштабирования.

#### 5. **Код: Извлечение логики фильтрации**

**Проблема:** Логика фильтрации находится прямо в компоненте, что усложняет тестирование.

**Рекомендация:** Вынести в отдельную функцию:

```typescript
function filterTodos(
  todos: Todo[],
  status: FilterStatus,
  searchText: string
): Todo[] {
  return todos
    .filter((todo) => {
      if (status === "active") return !todo.completed;
      if (status === "completed") return todo.completed;
      return true;
    })
    .filter((todo) =>
      todo.text.toLowerCase().includes(searchText.toLowerCase())
    );
}

// В компоненте:
const filteredTodos = useMemo(
  () => filterTodos(todos, filterStatus, searchText),
  [todos, filterStatus, searchText]
);
```

**Импакт:** Низкий - улучшает тестируемость и читаемость кода.

## 🔍 Технический долг и потенциальные проблемы

### 1. **Производительность при масштабировании**
При количестве задач > 1000, текущая реализация может показывать задержки:
- Две последовательные операции `.filter()` на каждый рендер
- `toLowerCase()` вызывается для каждой задачи на каждый символ в поиске

**Решение:** Реализовать виртуализацию списка (react-window) при необходимости.

### 2. **Отсутствие персистентности фильтров**
Фильтры сбрасываются при обновлении страницы.

**Решение:** Сохранять состояние фильтров в localStorage или URL query params.

### 3. **Локализация**
Текст кнопок и сообщений захардкожен на английском языке.

**Решение:** Добавить i18n (например, next-intl) для поддержки нескольких языков.
